🔴 What is NP-Completeness?▶

A problem is NP-Complete if it is both in NP and NP-Hard. This means it's one of the hardest problems in the class NP — if you could solve it efficiently, you could solve ALL NP problems efficiently.

To prove NP-Completeness you need TWO things:

1. Show the problem is in NP — given a solution, you can verify it quickly.
2. Show the problem is NP-Hard — reduce a known NP-Complete problem TO it in polynomial time.

✅ Step 1: Max-Cut is in NP▶
1
Certificate: A partition of vertices into two sets S and V\S.
2
Verifier: Given a certificate (S, V\S), count all edges with one endpoint in S and one in V\S. If count ≥ k, accept. This takes O(|E|) time — polynomial.
3
Conclusion: Since we can verify any solution in polynomial time, Max-Cut ∈ NP. ✓
🔁 Step 2: Reduction from NAE-3-SAT → Max-Cut▶

What is NAE-3-SAT?

Not-All-Equal 3-SAT: given a 3-CNF formula, is there an assignment where in every clause, the three literals are NOT all equal (not all true and not all false)? This is a known NP-Complete problem.

Construction: NAE-3-SAT → Max-Cut
1
Variables: For each variable xᵢ in the formula, create two nodes: xᵢ (true literal) and ¬xᵢ (false literal). Add an edge between them — this forces them into opposite sides of the cut.
2
Clauses: For each clause (a ∨ b ∨ c), add edges between all pairs: a-b, b-c, a-c. This creates a triangle on the three literal nodes of the clause.
3
Threshold: Set k = (number of variable edges) + (number of clauses × 2). A valid NAE-3-SAT assignment corresponds to a cut of exactly this size.
4
Polynomiality: The construction adds at most O(n) variable edges and O(m) clause triangles = O(n + m) total — polynomial in input size.
Example: clause (x₁ ∨ x₂ ∨ ¬x₃)

✓ Forward Direction (YES → YES)

If the NAE-3-SAT formula is satisfiable, place true literals in S and false literals in V\S. Since NAE means not all equal, each clause has at least one literal in each set → at least 2 edges of the triangle cross the cut. Total cut ≥ k.

✓ Backward Direction (YES → YES)

If the Max-Cut has size ≥ k, each clause triangle must contribute at least 2 crossing edges. This means at least one literal is in S and one in V\S → they have different truth values → clause is NAE-satisfied. The variable edges ensure xᵢ and ¬xᵢ are on opposite sides (consistent assignment).

✓ Polynomiality

Construction creates n variable edges (one per variable) and 3m clause edges (triangle per clause). Total O(n + m) edges and O(n + m) nodes — clearly polynomial in input size.

∴ Conclusion: Max-Cut is NP-Complete

Max-Cut ∈ NP (verified above) AND NAE-3-SAT ≤ₚ Max-Cut (reduction above). Therefore Max-Cut is NP-Complete. No polynomial-time algorithm is known unless P = NP.

⚖️ Brute Force vs Greedy Local Search▶

💥 Brute Force

• Try ALL 2ⁿ partitions
• Guarantees optimal solution
• Time: O(2ⁿ · |E|)
• Usable up to ~20 nodes
• Completely impractical for large graphs

🔍 Greedy Local Search

• Start random, flip vertices greedily
• Guaranteed ≥ |E|/2 cut (0.5-approx)
• Time: O(|V| · |E|)
• Practical for large graphs
• May miss global optimum
Operations growth comparison
Brute Force O(2ⁿ·|E|)
Local Search O(|V|·|E|)
📐 0.5-Approximation Guarantee Proof▶
1
Local Maximum Definition: A partition is a local maximum if no single vertex flip increases the cut size.
2
Key Observation: At a local maximum, for every vertex v, moving v to the other side does NOT increase the cut. This means: the number of edges from v crossing the cut ≥ edges from v NOT crossing the cut.
3
Formal argument: For each vertex v with degree d(v): at least d(v)/2 edges must cross the cut (otherwise flipping v would improve). Summing over all vertices: 2|CUT| ≥ Σ d(v)/2 · 2 = |E|
4
Conclusion: |CUT| ≥ |E|/2. Since the optimal cut OPT ≤ |E|, we get |CUT| ≥ OPT/2 — a 0.5-approximation! ✓

✓ What this means practically

The greedy local search always finds a cut that is AT LEAST half the maximum possible. For example, if the optimal cut has 100 edges, greedy finds at least 50. The famous Goemans-Williamson algorithm achieves 0.878 using semidefinite programming, but local search is much simpler and still provides this guarantee.

✗ Counterexample: Local ≠ Global Optimum

Consider a cycle C₄: nodes 0-1-2-3-0. Local search might find cut {0,2} vs {1,3} = 4 edges (optimal). But if it starts with {0,1} vs {2,3} = 2 edges and no flip helps locally, it could stop at a non-optimal local maximum. This shows greedy can miss the global optimum.

📖 What is Max-Cut?▶

Given an undirected graph G = (V, E), partition the vertices into two sets S and V\S to maximize the number of edges with one endpoint in each set. These edges are called cut edges.

NP-Complete
No poly-time exact algorithm known
0.878 approx
Best known: Goemans-Williamson
0.5 approx
Simple greedy local search
🧠 Why Greedy? Design Paradigm Justification▶

Max-Cut uses the Greedy Local Search paradigm. Here is why this paradigm fits, and why other paradigms do not.

✓ Why Greedy Local Search fits Max-Cut

The greedy choice is: if moving one vertex to the other side increases the cut, move it. This is locally optimal and provably moves toward a better solution. Each flip strictly increases the cut size, so the algorithm always terminates. At a local maximum, the 0.5-approximation guarantee holds.

✗ Why Dynamic Programming does NOT fit

Dynamic Programming requires optimal substructure — the optimal solution to the full problem must be built from optimal solutions to subproblems. Max-Cut does not have this property. The best cut for a subgraph tells us nothing reliable about the best cut for the full graph, because adding more nodes changes which partition is optimal in unpredictable ways.

✗ Why Divide and Conquer does NOT fit

Divide and Conquer splits the problem into independent subproblems, solves each, then combines results. For Max-Cut, if we split the graph into two halves and find the best cut for each half independently, we cannot combine them into the best cut for the whole graph — the two halves interact through edges that connect them, and those cross-edges are ignored when solving independently.

✓ Design Summary

Paradigm: Greedy Local Search
Greedy choice: flip a vertex if it strictly increases the cut
Termination: each flip increases cut by ≥ 1, cut ≤ |E|, so at most |E| flips
Correctness: at local maximum, cut ≥ |E|/2 (0.5-approximation)
Time complexity: O(|V| · |E|) — polynomial

🔍 Greedy Local Search Algorithm▶

Algorithm Steps

1. Randomly assign each vertex to S or V\S.
2. For each vertex v: compute gain = (edges from v NOT crossing) - (edges from v crossing).
3. If gain > 0, flip v to the other side (this increases cut size by gain).
4. Repeat until no vertex can be flipped to improve the cut.
5. Return the current partition — this is a local maximum.

✓ Why it always terminates

Each flip strictly increases the cut size. Since the cut size is bounded above by |E|, the algorithm terminates in at most |E| flips. Total time: O(|V| · |E|).

✗ Real-world applications

Circuit design: Minimize wire crossings between chips.
Physics: Spin glass models — minimize energy of magnetic interactions.
Network design: Partition network for load balancing.
VLSI layout: Split components across layers.

💥 Brute Force Algorithm▶

How it works

Enumerate all 2ⁿ possible subsets S of V. For each subset, count how many edges have exactly one endpoint in S. Return the subset with maximum count. Guaranteed optimal but exponential time.

🔍 Worked Example: Greedy on Cycle C4▶

Graph: Cycle C4 — nodes {0,1,2,3}, edges {0-1, 1-2, 2-3, 3-0}. Optimal cut = 4 edges (alternating partition). Let's trace greedy local search step by step.

Step 1 — Initialize: Assign alternating sides
S = {0, 2}  |  V\S = {1, 3}
Cut edges: 0-1 ✂, 1-2 ✂, 2-3 ✂, 3-0 ✂
Cut size = 4 ← already optimal!

Now check each vertex: can any flip improve the cut?
Step 2 — Check vertex 0 (in S): should we flip?
Vertex 0 connects to: 1 (V\S = crossing ✓) and 3 (V\S = crossing ✓)
Crossing edges from 0: 2  |  Same-side edges from 0: 0
Gain from flipping = same_side - crossing = 0 - 2 = -2 (worse!)
→ Do NOT flip vertex 0.
Step 3 — Check vertices 1, 2, 3: all give negative gain
By symmetry of C4, every vertex has exactly 2 crossing edges and 0 same-side edges.
No vertex can be flipped to improve the cut.
→ LOCAL MAXIMUM REACHED! Cut = 4 = |E|/2 × 2 ✓

This is actually the OPTIMAL cut for C4. Greedy found it immediately from the alternating initialization.
⚠ Greedy Failure Example — Bad Initialization on K4
Graph K4 (complete graph, 6 edges). Optimal cut = 4 edges (split 2-2).
Bad start: S={0} V\S={1,2,3} → cut = 3 edges.
Check vertex 0: crossing=3, same=0, gain=-3 → no flip.
Check vertex 1: crossing=1(to 0), same=2(to 2,3), gain=2-1=1 → FLIP!
After flip: S={0,1} V\S={2,3} → cut = 4 edges = optimal ✓
Greedy recovered to optimal here, but this isn't always guaranteed.
